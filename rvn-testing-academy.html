Dynamic table with pagination and multi-column sorting

Hard

Automate a data table that loads rows dynamically via AJAX, supports multi-column sorting, and has server-side pagination. Validate data integrity across all pages after sorting by two columns simultaneously.

SeleniumJavaTestNG

Approach

Use explicit waits with ExpectedConditions.stalenessOf() to handle AJAX refreshes. Store the first-page data before sorting, then traverse all pages comparing sorted order.

Key hints

1. Extract table data into a List<List<String>> per page. 2. Handle StaleElementReferenceException with retry logic. 3. For multi-sort, check if Shift+Click is needed. 4. Compare Java-sorted list with extracted list to validate sort correctness.

// Wait for table to reload after sort
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.stalenessOf(firstRow));
wait.until(ExpectedConditions.visibilityOfElementLocated(tableLocator));

File upload with drag-and-drop validation

Hard

Build an automation solution for a drag-and-drop file uploader that enforces file type restrictions, max file size, and multi-file selection. Validate error messages for invalid uploads and success states.

PlaywrightTypeScript

Approach

Use Playwright's setInputFiles() for standard file inputs. For pure drag-and-drop without input, use page.dispatchEvent() to simulate DataTransfer. Test boundary cases: exact max size, max+1 byte, and disallowed extension.

// Playwright drag-and-drop file upload
const [fileChooser] = await Promise.all([
  page.waitForEvent('filechooser'),
  page.locator('[data-testid="drop-zone"]').click(),
]);
await fileChooser.setFiles('./test-data/large-file.pdf');

OAuth2 login flow with token refresh automation

Hard

Automate authentication through an OAuth2 provider, intercept access tokens, simulate token expiry, and verify the application correctly refreshes tokens without requiring re-login.

CypressAPI mockingJavaScript

Approach

Intercept the token endpoint with cy.intercept(). Store the access token in Cypress state. Override the token with an expired one by manipulating localStorage or cookies. Assert the app silently refreshes and continues functioning.

cy.intercept('POST', '/oauth/token', (req) => {
  req.reply({ statusCode: 401, body: { error: 'token_expired' } });
}).as('expiredToken');
// Then verify refresh happens automatically

End-to-end checkout with flaky elements

Medium

Test an e-commerce checkout flow where certain UI elements appear with random delays. Implement smart waits, retry strategies, and handle intermittent network failures without test flakiness.

SeleniumPythonpytest

Approach

Wrap flaky interactions in a retry decorator. Use WebDriverWait with custom conditions. Separate stable steps from flaky ones and apply different wait strategies per step type.

def retry(times=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    if attempt == times - 1:
                        raise e
        return wrapper
    return decorator

Visual regression testing across 4 breakpoints

Medium

Set up visual regression testing that captures and compares screenshots at mobile, tablet, desktop, and wide-screen breakpoints. Implement baseline updates on the pipeline.

PlaywrightPercyGitHub Actions

Approach

Use Playwright's viewport configuration to test each breakpoint. Use toMatchSnapshot() for local comparisons or Percy for cloud-based visual diffing with CI integration.

const viewports = [
  { width: 375, height: 812, name: 'mobile' },
  { width: 768, height: 1024, name: 'tablet' },
  { width: 1280, height: 800, name: 'desktop' },
  { width: 1920, height: 1080, name: 'wide' },
];
for (const vp of viewports) {
  await page.setViewportSize({ width: vp.width, height: vp.height });
  await expect(page).toHaveScreenshot(`homepage-${vp.name}.png`);
}

GraphQL API test automation suite

Medium

Build a complete test suite for a GraphQL API covering queries, mutations, subscriptions, error handling, and schema validation using automated tooling.

JestGraphQLTypeScript

Approach

Use graphql-request or Apollo Client for test requests. Validate response shapes against the schema using graphql-inspector. Test error cases by sending invalid variables and malformed queries.

const query = gql`
  query GetUser($id: ID!) {
    user(id: $id) {
      id name email role
    }
  }
`;
const data = await client.request(query, { id: '123' });
expect(data.user).toMatchObject({ id: '123' });

JMeter distributed load test with real-time monitoring

Beginner-friendly

Set up a JMeter test plan that simulates 500 concurrent users on a login + search flow, streams results to InfluxDB, and visualizes in Grafana in real-time.

JMeterInfluxDBGrafana

Approach

Create a Thread Group with 500 threads, 60s ramp-up, and a Loop Controller for duration. Add HTTP Samplers for each step. Configure InfluxDB Backend Listener. Import the JMeter Grafana dashboard (ID: 5496).

Key metrics to monitor: Active threads, response time percentiles (P50, P95, P99), error rate, and throughput (req/sec).
